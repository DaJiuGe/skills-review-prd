# 在线订餐系统 PRD

## 1. 项目背景

本系统面向 C 端消费者，提供在线选餐、下单、支付、配送追踪能力。

## 2. 目标用户

- 普通消费者：浏览菜单、下单、支付、查看订单状态。
- 餐厅管理员：管理菜单、查看订单。
- 配送员：接收配送任务、更新配送状态。

## 3. 核心流程

### 3.1 下单流程

用户选择菜品后生成订单（Order），订单包含配送地址和支付方式。用户点击提交按钮，系统执行 PlaceOrder 命令。订单创建成功后触发 OrderPlaced 事件。

### 3.2 配送流程

系统根据订单地址分配配送（Delivery）任务。配送员接单后，订单状态变为“配送中”。

### 3.3 支付流程

系统调用 PaymentGateway 完成支付。PaymentGateway 回调通知支付结果，若成功则产生 PaymentConfirmed 事件。

## 4. 用户界面

### 4.1 用户订单追踪页面

用户可在个人中心查看订单状态，该页面读取 Order 聚合投影生成的 OrderStatusView。

## 5. 非功能需求

- 峰值 QPS 不低于 1000。
- 支付失败后的订单状态流转规则需补充。
